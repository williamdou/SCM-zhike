# 开发与应用报告

**SCM智课——供应链管理研究生智能教学系统**

# 一、开发背景

"供应链管理研究"是面向管理科学与工程等专业研究生的核心课程，48学时，覆盖16个知识模块。实际教学中面临四个痛点：

**痛点一：课堂沉闷。**教学陷于固定模式，概念讲授占70%课时，主动发言者寥寥。

**痛点二：理论与实践脱节。**学生知道"牛鞭效应"的定义，却从未体验其在多级供应链中的逐级放大效应。供应链管理本质上是决策科学，但传统课堂剥夺了学生"做决策"的机会。

**痛点三：缺乏动手参与。**国际上啤酒游戏等体验式学习已是SCM教学标配，但国内课堂受限于技术和开发能力，教师讲学生听仍是主流。

**痛点四：学术视野狭窄。**学生对学术流派、研究脉络、方法论缺乏系统认知。

核心矛盾：如何在48学时内，既完成16章知识体系教学，又让每章至少有1课时动手实践，还能融合企业案例与学术视野？必须借助生成式人工智能重构教学内容生产方式和课堂组织方式。

# 二、设计与开发

## （一）平台/技术选择

核心设计原则为"零代码、云端运行、课堂即开即用"。经综合比较国内主流AI平台，选择Dify Cloud（公有云版）作为开发和运行平台：

| 考量维度 | Dify Cloud优势 |
|---|---|
| 开发门槛 | 可视化知识库+拖拽式Chatflow，无需编写代码 |
| 大模型接入 | 原生支持DeepSeek API，符合国产大模型要求 |
| 知识库能力 | 多知识库并行、多格式文档导入、自动向量化与混合检索 |
| 对话能力 | Chatflow支持多轮对话、条件分支、变量状态管理 |
| 发布方式 | 一键发布为公开Web应用，生成固定URL |
| 成本 | 免费版已满足课程级应用规模 |

技术架构：DeepSeek API（大模型服务）+ Dify Cloud（应用平台）+ Dify内置向量存储与混合检索（知识库引擎）+ Chatflow多分支工作流 + Markdown/Mermaid结构化输出。

## （二）开发过程

开发过程体现"借助AI开发AI"核心理念。作为管理学背景教师，关键环节均借助DeepSeek辅助完成：

**阶段一：16章四维全景映射构建。**将16个章节逐一输入DeepSeek，使用结构化提示词生成每章的四维教学资源映射表。以"库存管理"章为例的提示词：

> "请为'库存管理'章构建四维教学资源映射表：(1)企业实践维度：2个典型企业案例（含核心做法与数据）；(2)学术研究维度：学术流派、研究脉络、近3年热点、5篇经典文献（含期刊名、年份、核心论点、理论框架、研究方法和结论）；(3)职业发展维度：2个对标岗位的JD、技能、认证、薪资；(4)互动实践维度：1个基于真实数据的决策模拟练习（含情景数据、决策变量、评价规则）。可使用啤酒游戏的数字化变体。"

DeepSeek输出后笔者逐条审核修正，确保案例数据准确、文献真实、岗位JD可信、任务难度合理。

**阶段二：四大知识库构建。**在Dify Cloud中创建四个独立知识库：

| 知识库 | 内容来源 | 规模 |
|---|---|---|
| 企业实践案例库 | 36氪/虎嗅/公司年报等公开报道；DeepSeek辅助摘要 | 约40个案例 |
| 学术研究知识库 | SCM领域近10年高被引论文；经典教材核心理论 | 约80篇文献 |
| 职业发展信息库 | 猎聘/Boss直聘/LinkedIn岗位JD；行业研报 | 32个岗位 |
| 互动实践题库 | 每章1-2个实践任务的情景数据、规则、参考答案 | 20余个任务 |

Dify自动向量化后支持混合检索。

**阶段三：Chatflow双模式工作流编排。**模式A（内容展示）：教师输入章节名→系统并行检索四个知识库→调用DeepSeek按预设Prompt模板组织为四维Markdown输出→投屏展示。模式B（互动实践）：系统加载该章实践任务情景数据→引导式多轮对话：展示问题背景与数据→学生输入决策→系统计算反馈→追问/变体挑战→总结概念关联。以啤酒游戏为例：5轮决策后展示订货量、需求、库存三条波动曲线，分析牛鞭效应生成机制，关联"信息共享""需求预测更新""安全库存"概念。Prompt模板借助DeepSeek反复迭代优化。

**阶段四：发布与课堂迭代。**调试完成后一键发布。选取2章试讲优化后，在全学期应用。

AI赋能证据：①内容初始生成由DeepSeek完成；②40余个案例由DeepSeek提取结构化；③20余个互动任务由DeepSeek辅助设计；④所有Prompt模板借助DeepSeek迭代优化；⑤全程零编程，代码由DeepSeek辅助生成。

## （三）功能架构

系统核心功能围绕"一个入口、双模切换、四维联动"：

（1）**教师交互界面**：Dify公开Web App URL，浏览器直接访问，输入章节名即可使用。

（2）**企业实践引擎**：按章节匹配企业案例及B站等富媒体链接，AI一键生成讨论题。

（3）**学术研究图谱**：学术流派分类、研究脉络Mermaid图、热点趋势、必读文献推荐。

（4）**职业发展资源仓**：32个岗位对标（JD、技能、认证、薪资），B站视频链接。

（5）**互动实践引擎**：16章每章至少1个基于真实数据的决策模拟任务：

| 章节 | 实践任务 | 互动形式 |
|---|---|---|
| 供应链概念 | 绘制供应链全景地图 | 识别7环节→系统评估完整性 |
| 供应链产生逻辑 | 比亚迪vs苹果：一体化还是外包 | 三维度分析→展示交易成本框架 |
| 发展趋势 | 技术趋势评估矩阵 | 6技术评估→系统排名与理由 |
| 战略匹配 | Fisher矩阵实战匹配 | 4产品选供应链类型→系统校验 |
| 供应链整合 | 信息共享实验 | 对比有/无共享情境的绩效差异 |
| 供应链网络 | 设施选址决策 | 3候选位置→选最优→与MIP比较 |
| 成员选择与协调 | 供应商多准则评估 | 5供应商设权重→系统排名 |
| 产品设计与研发 | 延迟策略设计 | 给定产品设计推拉边界位置 |
| 产品定价 | 需求-价格弹性模拟 | 多轮试探→绘制价格-利润曲线 |
| 外包与采购管理 | 自制vs外购决策 | 盈亏平衡计算→系统校验纠错 |
| 库存管理 | 啤酒游戏数字化版 | 5轮回合制→成本曲线→牛鞭效应 |
| 物流配送管理 | 配送路线优化 | 5配送点规划路线→与最优解对比 |
| 信息价值 | 信息共享价值量化 | 分角色输入→汇总牛鞭效应放大程度 |
| 生产计划 | 主生产排程挑战 | 需求+MPS→排程→评估产能利用率 |
| 风险管理 | 供应链中断应急 | 情景展示→预案选择→恢复时间评估 |
| 绩效管理 | SCOR KPI体系设计 | 给定企业设计KPI→系统对标反馈 |

（6）**课堂反馈收集**：嵌入腾讯问卷，学生扫码匿名评价。

# 三、应用过程与效果

## （一）课程整体设计

课程共48学时、16次课。每次课教学结构：

| 时间 | 环节 | 系统模式 | 教学活动 |
|---|---|---|---|
| 第1学时： | 案例导入+核心概念 | 内容展示 | 系统投屏案例与学术脉络，教师精讲核心概念 |
| 第2学时 | 互动动手实践 | 互动实践 | 学生分组讨论后输入决策，系统实时反馈，各组对比 |
| 第3学时 | 学术升华+职业拓展 | 内容展示 | 系统展示学术维度与职业对标，引导实践升华为学术认知 |

概念讲授压缩至约25%，每章至少1学时动手实践。

## （二）典型课堂场景——以"库存管理"章为例

**第1学时：**：系统投屏企业案例（Zara库存周转12次/年 vs H&M 4次/年）与学术脉络（EOQ→报童→多级库存→VMI→CPFR），展示"库存规划分析师"岗位对标。

**第2学时**：启动啤酒游戏。学生分组做5轮订货决策，系统每轮反馈库存与成本，总结轮展示曲线对比，牛鞭效应一目了然。

**第3学时**：系统展示Lee, Padmanabhan & Whang (1997)经典文献，"你们刚体验的牛鞭效应，从学术角度可通过信息共享、缩短提前期、减小批量三种机制缓解。哪组的数据验证了哪种机制最有效？"

## （三）应用效果

经完整学期试点：

| 指标 | 往届同期 | 本学期 |
|---|---|---|
| 单次课学生主动发言/操作人次 | 2-3 | 15-20 |
| 课堂参与度（自评1-5） | 3.0 | 4.5 |
| 期末评教满意度（1-5） | 4.0 | 4.7 |
| 核心概念理解深度 | 能复述但应用弱 | 能结合实践阐述 |
| 课程论文与企业实践结合度 | 25% | 72% |
| 论文文献综述有学术流派意识 | 30% | 82% |

**典型学生反馈**："啤酒游戏那节课是我上过最好的研究生课。自己做了几轮决策看到库存忽高忽低成本飙升，一下子就懂了。""供应商评估练习让我真正理解了多准则决策，自己动手调整权重才知道权重设置多关键。""每章都能了解对应企业和岗位，特别实用。"

# 四、创新与反思

## （一）创新点

**1. 四位一体课堂内容组织模式。**将企业实践、学术研究、职业发展、互动实践在每次课中即时融合，形成"看案例→做决策→悟概念→明方向"的完整学习闭环。

**2. 基于AI的游戏化互动实践引擎。**利用Dify Chatflow多轮对话和状态管理能力，在16章实现了游戏化互动。学生不再是"听"供应链管理，而是"做"供应链管理，从概念记忆升级为决策体验。

**3. 零代码开发的教师AI自主实践典范。**作为管理学背景教师，全程未编程，仅通过Dify可视化界面和DeepSeek辅助，独立完成涵盖16章、20+互动任务的复杂系统开发。知识库文档由DeepSeek结构化输出，Prompt由DeepSeek迭代优化，情景数据由DeepSeek辅助设计。这本身就是"AI赋能教师突破技术壁垒"的典型范例。

**4. 48学时系统性重构而非单点改良。**系统覆盖全部16章，为同类课程提供了可参考的完整范式。

## （二）问题与反思

**1. 企业案例时效性挑战。**改进：建立季度更新机制。

**2. 学术深度与趣味性平衡。**已增设"学术升华"环节，教师用3-5分钟将讨论链接到理论框架。

**3. 互动任务计算复杂性。**部分任务计算逻辑复杂，改进方向：接入Dify代码执行节点。

**4. 覆盖面扩展。**后续计划：开源Dify工作流模板和Prompt库至Gitee；撰写面向非技术教师的部署指南；在学院推介覆盖更多课程。

**5. 富媒体素材版权。**改进：使用企业官方内容，关键素材本地备份。
